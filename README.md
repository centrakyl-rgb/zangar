# Telegram autopost

Простая система автопостинга в Telegram-канал через GitHub Actions.

## Как это работает

1. Ты пишешь посты в `posts.md`.
2. GitHub Actions запускает `scripts/autopost.py` по расписанию.
3. Скрипт смотрит в `state.json`, какие посты уже были опубликованы.
4. Скрипт берёт следующий неопубликованный пост и отправляет его в Telegram.
5. После успешной отправки скрипт обновляет `state.json`.
6. GitHub Actions сохраняет новый `state.json` обратно в репозиторий.

Так посты не повторяются.

## Файлы проекта

- `posts.md` — очередь постов.
- `state.json` — память системы: какие посты уже вышли.
- `scripts/autopost.py` — скрипт публикации.
- `.github/workflows/telegram-autopost.yml` — расписание GitHub Actions.
- `requirements.txt` — зависимости Python.
- `.env.example` — пример локальных настроек.

## Формат постов

```md
### Пост 1
Заголовок

Текст поста.

Источник / подпись.
```

Если нужен пост с картинкой, добавь строку `image:` в начало поста:

```md
### Пост 2
image: images/post-2.jpg

Заголовок

Текст поста.
```

Картинку положи в папку `images`.

## Секреты GitHub

В репозитории открой:

`Settings` → `Secrets and variables` → `Actions` → `New repository secret`

Добавь два секрета:

- `TELEGRAM_BOT_TOKEN`
- `TELEGRAM_CHAT_ID`

Токен нельзя хранить в коде.

## Как получить TELEGRAM_BOT_TOKEN

1. Открой Telegram.
2. Найди бота `@BotFather`.
3. Напиши `/newbot`.
4. Придумай имя бота.
5. Придумай username бота, он должен заканчиваться на `bot`.
6. BotFather выдаст токен.
7. Сохрани токен в GitHub Secret `TELEGRAM_BOT_TOKEN`.

## Как получить TELEGRAM_CHAT_ID

Самый простой вариант для публичного канала:

1. Добавь бота в администраторы канала.
2. Дай ему право публиковать сообщения.
3. Используй username канала как `TELEGRAM_CHAT_ID`.

Пример:

```txt
@my_channel
```

Если канал приватный, обычно нужен числовой id вида:

```txt
-1001234567890
```

Простой способ получить его:

1. Добавь бота в канал администратором.
2. Опубликуй любой тестовый пост в канал.
3. Перешли этот пост боту `@userinfobot` или похожему боту для просмотра id.
4. Используй id канала в `TELEGRAM_CHAT_ID`.

## Локальный тест

Скопируй `.env.example` в `.env` и заполни значения.

Посмотреть следующий пост без отправки:

```bash
python scripts/autopost.py --dry-run
```

Отправить один пост:

```bash
python scripts/autopost.py
```

После успешной отправки `state.json` обновится.

## Проверка ошибок

Скрипт показывает понятные ошибки, если:

- файл `posts.md` не найден;
- токен Telegram неправильный;
- `TELEGRAM_CHAT_ID` неправильный;
- бот не добавлен в администраторы канала;
- картинка для поста не найдена;
- все посты уже опубликованы.

## Как пользоваться дальше

1. Добавляй новые посты в конец `posts.md`.
2. Не удаляй `state.json`, если не хочешь повторных публикаций.
3. Если нужно изменить расписание, открой `.github/workflows/telegram-autopost.yml` и поменяй строку `cron`.
4. Для ручного запуска открой вкладку `Actions`, выбери `Telegram autopost` и нажми `Run workflow`.
